# Curriculum map

## Beginner

Purpose: build the minimum viable AI mental model.

- AI, machine learning, deep learning, generative AI
- Data, model, training, inference
- Supervised, unsupervised, reinforcement learning
- Classification, regression, overfitting, data splits
- LLM, token, context window, temperature, hallucination
- Embeddings, semantic search, vector databases
- Prompt anatomy, zero-shot, few-shot, output format
- Bias, fairness, human oversight
- Core tools: ChatGPT, Claude, Gemini, Perplexity, NotebookLM, GitHub Copilot, Codex

## Intermediate

Purpose: become an AI operator who can use and specify workflows.

- Transformers, attention, parameters, pretraining, instruction tuning, RLHF, DPO
- Fine-tuning, LoRA, quantization, multimodal models
- RAG, document chunks, overlap, hybrid search, reranking, metadata filtering, citations
- Agents, tool calling, memory, planner-executor, ReAct, human-in-the-loop
- Prompt injection and prompt safety
- APIs and frameworks: OpenAI API, Anthropic API, Google AI Studio, LangChain, LangGraph, LlamaIndex
- Evaluation: precision, recall, F1, confusion matrix, golden datasets, LLM-as-judge, rubrics
- Math: loss, gradient descent, probability

## Advanced

Purpose: build reliable AI systems.

- RAG failure modes and retrieval evals
- Knowledge graphs and GraphRAG-style thinking
- Tool permissions, least privilege, idempotency, state machines
- MCP-style tool/data connections
- Defensive AI-worm awareness
- Production concerns: latency, cost, routing, fallbacks, rate limits, retries, caching
- Privacy, governance, red teaming, observability
- Tools: Docker, FastAPI, Streamlit, PyTorch, scikit-learn
- Embedding math: cosine similarity, dimensionality, calibration

## Expert

Purpose: think like an AI system/product strategist.

- Foundation models, scaling laws, mixture of experts, knowledge distillation
- Synthetic data, model cards, alignment
- Open vs closed model tradeoffs
- Eval-first development
- AI product wedges
- Human-AI interface design

## Capstone prompt

Design one AI workflow for a real problem. Include:

1. User problem and success metric.
2. AI task boundary.
3. Model/tool choices.
4. Data and retrieval plan.
5. Prompt/context design.
6. Human approval points.
7. Evaluation dataset and rubric.
8. Safety/privacy controls.
9. Failure modes and fallback behavior.
10. Deployment and monitoring plan.
